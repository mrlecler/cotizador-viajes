1. Logo — Wordmark ermix
✓ Uso correcto
• El nombre siempre en minúscula: ermix
• El logo se construye dinámicamente con buildWordmark() en js/app-core.js
• ⚠️ La X custom mantiene su forma y path SVG exactos — NO modificar
• Solo cambian los hexadecimales del gradiente dentro de buildWordmark():
  #4F46E5 → #1B9E8F
  #7C3AED → #0BC5B8
  #F43F5E → #06B6D4
• LEER el código de buildWordmark() antes de hacer cualquier cambio
✗ Uso incorrecto
• Nunca escribir "Ermix", "ERMIX" o "ErMix"
• Nunca usar el gradiente violeta-coral del v2
• Nunca modificar el path SVG de la X custom
• Nunca separar el logo en partes
• Nunca colocar el logo del agente en el header de la app
• El logo del agente solo va en el PDF de cotización
• Nunca usar emojis como sustituto del logo

10. Reglas absolutas de desarrollo
✓ Siempre hacer esto
• Usar variables CSS (var(--primary), etc.) — nunca hardcodear
• SVG Lucide para todos los íconos, stroke-width 1.8, stroke-linecap round
• El gradiente oficial: var(--grad)
• Photo headers en todas las secciones del formulario y vista del cliente
• Pasteles por tipo de sección en cards y recuadros de precio
• Sidebar lateral de 54px con margin-left: 54px en el contenido
• DM Sans para todo el texto, DM Mono solo para códigos y valores técnicos
• data-theme="light" por defecto, data-theme="dark" como opción
✗ Nunca hacer esto
• Emojis en ningún lugar de la UI
• Hardcodear colores (no usar #7C3AED directo — usar var(--primary))
• Tocar la lógica de Supabase auth o guardado de cotizaciones
• Tocar window.print() o el sistema del PDF
• Tocar el sistema buildWordmark() (solo adaptar colores)
• Usar npm, bundlers o importar librerías externas pesadas
• Escribir "Ermix" con mayúscula — siempre "ermix"
• Usar el menú horizontal superior (#nav) — está eliminado en v3
